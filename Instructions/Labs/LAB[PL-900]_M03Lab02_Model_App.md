# 랩 3: 모델 기반 앱 빌드 방법

**WWL 테넌트 - 사용 약관** 강사 주도 교육 제공의 일부로 테넌트를 제공하는 경우 강사 주도 교육에서 실습 랩을 지원하기 위해 테넌트를 사용할 수 있습니다. 테넌트를 실습 랩 외부에서 공유하거나 사용해서는 안 됩니다. 이 과정에서 사용되는 테넌트는 평가판 테넌트이며 클래스가 종료된 후 사용하거나 액세스할 수 없으며 확장판에서도 사용할 수 없습니다. 테넌트를 유료 구독으로 변환해서는 안 됩니다. 이 과정의 일부로 얻은 테넌트는 Microsoft Corporation의 재산으로 유지되며 언제든지 액세스 권한을 획득하고 다시 소유할 수 있는 권리를 보유합니다. 

## 시나리오

Bellows College는 캠퍼스 내에 여러 건물이 있는 교육 기관입니다. 캠퍼스 방문자는 현재 종이 저널에 기록되어 있습니다. 이 정보는 일관되게 수집되지 않으며, 전체 캠퍼스 방문 데이터를 수집하고 분석할 방법이 없습니다.

캠퍼스 관리부는 건물 액세스가 보안 요원에 의해 제어되고, 모든 방문이 반드시 호스트에 의해 사전 등록 및 기록되는 현대화된 방문자 등록 시스템을 원합니다.

이 과정 전반에 걸쳐 벨로즈 칼리지 관리 및 보안 담당자가 캠퍼스 내 건물에 대한 액세스를 관리하고 제어할 수 있도록 애플리케이션을 빌드하고 자동화를 수행합니다.

이 랩에서는 백오피스 캠퍼스 직원이 전체 캠퍼스의 방문 레코드를 관리할 수 있도록 Power Apps 모델 기반 앱을 빌드합니다.

고급 랩 단계

모델 기반 앱을 만드는 과정에서 다음을 완료합니다.

- Bellows Campus Management로 이름이 지정된 새 모델 기반 앱 만들기

- 앱 탐색을 편집하여 필요한 테이블을 참조합니다.

- 앱에 필요한 테이블의 양식 및 뷰 사용자 지정

다음과 같은 구성 요소와 함께 작업합니다.

- **뷰**: 뷰를 사용하면 사용자가 양식 테이블에 기존 데이터를 표시할 수 있습니다.

- **양식**: 여기서 사용자는 테이블에서 새 행을 만들고 업데이트합니다.

둘 다 더 나은 사용자 경험을 위해 모델 기반 앱에 통합됩니다.

필수 조건

- **모듈 0 랩 0 - 랩 환경 검증** 완료

- **모듈 2 랩 1 - 데이터 모델링** 완료

시작하기 전에 고려해야 할 사항

- 사용자 환경을 개선하기 위해 어떤 사항을 변경해야 합니까?

- 빌드한 데이터 모델을 기반으로 모델 기반 앱에 무엇을 포함해야 합니까?

- 모델 기반 앱의 사이트맵에서 어떤 사용자 지정을 할 수 있습니까?

연습 1: 보기 및 양식 사용자 지정

**목표:** 이 연습에서는 모델 기반 앱에서 사용할 사용자 지정 생성 테이블의 뷰 및 양식을 사용자 지정합니다.

작업 #1: 방문 양식 편집

1.  아직 로그인하지 않은 경우 <https://make.powerapps.com>에 로그인합니다.

2.  아직 선택되지 않은 경우 오른쪽 상단에 있는 **[내 이니셜] 연습** 환경을 선택합니다.

3.  왼쪽의 탐색을 사용하여 **테이블을** 선택하고 **방문** 테이블을 엽니다.

    방문 테이블이 표시되지 않으면 올바른 환경에 있는지 확인합니다(2단계).

4.  **데이터 환경** 섹션에서 **양식을** 선택하고 **기본** 양식 형식의 **정보** 양식을 엽니다. (**중요:** 양식 형식이 **Main**인 양식을 선택해야 합니다.) 

    **중요:** 기본적으로 모든 양식은 정보이므로 선택한 양식이 **기본** 양식 형식이고 다른 것이 아닌지 확인합니다. 기본적으로 양식에는 이름 및 소유자라는 두 개의 필드가 있습니다.

5.  **속성** 패널의 화면 오른쪽에서 **표시 이름** 필드를 선택하고 다음으로 변경합니다.`Main Information`

6.  왼쪽 탐색 창에서 **테이블 열을** 선택하고 열을 폼으로 끌거나 열 이름을 클릭하여 **소유자** 필드 아래에 다음 필드를 추가합니다.

    1. **방문자**

    2. **Scheduled Start**

    3. **예약된 종료**

    4. **실제 시작**

    5. **실제 종료**

7.  **코드** 열을 드래그하여 양식 헤더에 놓습니다.

    헤더는 양식의 오른쪽 상단 영역입니다. 양식의 필드를 보려면 화면 오른쪽에 있는 속성 패널을 축소해야 할 수 있습니다.

8.  **코**드 필드가 선택된 상태에서 화면 오른쪽 속성 패널의 **읽기 전용**에 대한 체크박스를 선택합니다.

9.  **소유자** 필드를 선택합니다. 속성 패널에서 **레이블** 을 로 변경합니다. `Host`

10. 오른쪽 위에 있는 **저장 및 게시** 단추를 선택하고 저장 및 게시가 완료되기를 기다립니다.

11. 편집 보기가 새 브라우저 탭 또는 창에서 열린 경우 닫습니다. 그렇지 않으면 화면 왼쪽 위에서 **🡠 뒤로** 를 선택합니다. 이제 **방문** 테이블 양식으로 돌아가야 합니다.

12. 왼쪽 위에 있는 이동 경로 사용(**테이블** > **폼 방문** > ).**** **방문을** 선택하여 테이블 **속성 방문** 화면으로 돌아갑니다. 


작업 #2: 활성 방문 뷰 편집

이 작업에서는 기본 활성 방문 보기를 수정하고 오늘의 방문에 대한 새 보기를 만듭니다.

1.  **데이터 환경** 섹션에서 **보기를** 선택하고 **활성 방문** 보기를 엽니다.

2.  다음 필드를 클릭하거나 끌어다 놓아 다음 필드를 뷰에 추가합니다.

    1. **코드**

    2. **방문자**

    3. **Scheduled Start**

    4. **예약된 종료**

3.  **만든** 위치 열에서 드롭다운 메뉴를 선택하고 **제거**를 선택합니다. 이제 **만든** 필드가 보기에서 제거됩니다.

4.  개별 열 너비의 크기를 데이터에 맞게 조정합니다.

5.  **정렬 기준 ...** X를 선택하여 **이름을** 제거하고 대신 **예약된 시작을** 선택합니다.

6.  **예약된 시작을** 선택하여 정렬 순서를 최신 항목으로 변경**합니다**.

7.  오른쪽 위에서 **저장 및 게시** 단추를 선택하고 게시가 완료되기를 기다립니다.


작업 #3: 오늘의 방문에 대한 새 보기 만들기

이제 보기를 복제하여 오늘의 방문에 대한 새로운 보기를 만듭니다.

중요: 활성 방문 보기를 닫지 않는지 확인합니다. 이 보기를 활용하여 새 오늘의 방문 보기를 만듭니다.

1.  **다른 이름으로 저장**을 선택합니다.

2.  **이름을** 로 `Today’s Visits` 변경하고 **저장**을 선택합니다.

3.  속성 패널에서 **필터 편집**을 선택합니다.

4.  **+ 추가**를 선택하고 **행 추가**를 선택합니다.

5.  **예약된 시작을** 필드로 선택한 다음, 드롭다운의 조건으로 **** **오늘로 같**음 을 변경합니다.

6.  ...를 선택합니다 **.** **상태** 행에 **대한 추가 명령**과 **삭제**를 선택하여 해당 필터 조건을 삭제합니다.

7.  **확인을** 선택하여 조건을 저장합니다. 이제 예약된 시작 날짜가 현재인 레코드만 표시하도록 보기가 필터링 **됩니다** .

8.  뷰에 **실제 시작** 및 **실제 종료** 필드를 추가합니다.

    **참고:** 더 이상 보기 상태를 필터링하지 않으므로 완료된 보기를 포함하여 오늘의 모든 방문을 받게 됩니다. 이 필드는 완료된 방문 및 진행 중인 방문을 구별하는 데 도움이 됩니다.

9.  오른쪽 위에서 **저장 및 게시** 단추를 선택하고 게시가 완료되기를 기다립니다.


연습 2: 모델 기반 앱 만들기

**목표:** 이 연습에서는 모델 기반 앱을 만들고, 사이트 맵을 사용자 지정하고, 앱을 테스트합니다.

단순성과 시간을 위해 이 랩에서 일부 방문 열은 다루지 않습니다.

작업 #1: 앱 만들기

1.  로그인하지 않은 경우 에 <https://make.powerapps.com>로그인합니다.

2.  아직 선택되지 않은 경우 오른쪽 상단에 있는 **[내 이니셜] 연습** 환경을 선택합니다.

3.  왼쪽 탐색 영역에서 **+ 만들기** 를 선택합니다.

4.  모델 기반 애플리케이션 만들기:

    1. 앱 만들기 화면의 **시작 섹션에서** **빈** **앱을** 선택합니다.

    2. **Dataverse를 기반으로 하는 빈 앱**에서 **만들기**를 선택합니다.

    3. **이름**에 를 입력 `Bellows Campus Management` 하고 **만들기**를 선택합니다.

5.  새 모델 기반 애플리케이션이 로드된 후 **+ 페이지 추가** 단추를 선택합니다.

6.  **페이지 추가** 화면에서 **Dataverse 테이블을** 선택한 다음 **, 다음** 단추를 선택합니다.

7.  다음 테이블을 선택합니다.

    1. 방문

    2. 연락처

8.  두 테이블이 모두 있으면 **추가**를 선택합니다.

9.  화면 왼쪽의 탐색을 사용하여 **탐색**을 선택합니다.

10. 탐색 창에서 탐색이라고 표시된 아래 **의 새 그룹을** 선택합니다. 왼쪽에 있는 메뉴를 확장해야 할 수도 있습니다.

11. 화면 오른쪽의 **표시 옵션** 섹션에서 **Title** 속성을 로 변경합니다. `Security`

12. **저장**을 선택하고 변경 내용이 저장될 때까지 기다립니다.

13. **저장**이 완료되면 **게시** 단추를 선택하여 변경 내용을 게시합니다.


작업 #2: 테스트 앱

애플리케이션 시작

1. **재생** 단추를 선택하면 모델 기반 앱이 새 탭에 로드됩니다.

새 연락처 만들기

2.  앱에서 **내 활성 연락처** 보기를 열어야 합니다. 그렇지 않은 경우 왼쪽 탐색 영역에서 **연락처** 를 선택합니다.

3.  명령 모음에서 **+ 새로** 만들기를 선택합니다.

4.  **이름을** 로 입력하고 **성을** 로 `John` 입력합니다.`Doe`

5.  개인 이메일을 **전자 메일**에 입력합니다. 이 기능은 전자 메일을 받게 될 향후 랩에서 사용됩니다.

6.  **저장 후 닫기**를 선택합니다.

7.  이제 **내 활성 연락처** 보기에 작성된 연락처가 표시될 것입니다.

새 방문 만들기

8.  왼쪽 탐색에서 **방문을** 선택합니다(사이트맵이라고도 함).

9.  **+새로 만들기**를 선택합니다.

10. 다음과 같이 필드를 입력합니다.

    1. **이름**: `New test visit`

    2. **방문자**: **John Doe** 선택

    3. **예약된 시작**: 내일 날짜와 오후 2:00를 시작 시간으로 선택합니다.

    4. **예약된 종료**: 내일 날짜 및 종료 시간으로 오후 3시 30분 선택

11. **저장 후 닫기**를 선택합니다. 그러면 방문이 만들어지고 **활성 방문** 보기에서 볼 수 있습니다.

12. 활성 **방문** 옆에 있는 드롭다운을 사용하여 오늘 방문으로 보기를 변경 **합니다**. 내일로 예정되어 있기 때문에 더 이상 뷰에서 새 방문을 볼 수 없습니다.

13. 테스트 레코드를 더 추가할 수 있습니다.
